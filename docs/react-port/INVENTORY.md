# Tweeq Codebase Inventory (for the React port)

> Generated 2026-07-12 by a full survey of `src/`. Companion to `PLAN.md` / `CONVENTIONS.md`.
> Goal: let a fresh agent port any given component **without re-reading the whole codebase**.
> All paths relative to repo root. LOC counts are whole-directory (template + script + style).

Legend for the "Vue surface" column:
`M` = `defineModel` (v-model) ┬Ě `M:x` = named model (`v-model:x`) ┬Ě `P` = props ┬Ě `E` = emits ┬Ě
`S` = slots ┬Ě `Sc` = scoped slot ┬Ě `X` = `defineExpose` ┬Ě `PI` = provide/inject ┬Ě `T` = Teleport ┬Ě
`Tr` = `<Transition>` ┬Ě `pop` = native Popover API (`popover` attr / `showPopover`) ┬Ě
`anchor` = CSS Anchor Positioning via `addAnchorName` (src/util.ts:86)

## 7. Risks & gotchas

1. **Native Popover API + CSS Anchor Positioning everywhere** (Popover, Tooltip, MultiSelectPopup, PaneModal, CommandPalette, TweakOverlay). Positioning is done with `anchor()` insets + `position-try-fallbacks` (Popover.vue:183ÔÇô242) plus a JS cross-axis shift; anchor names are refcounted via `addAnchorName` (util.ts:60). This is Chromium-baseline tech ÔÇö no floating-ui in the codebase anymore; do NOT reintroduce it. Exit animations rely on `@starting-style` + `transition: display/overlay ... allow-discrete`. React must call `togglePopover()` in effects after mount and listen to the native `toggle` event to sync state.
2. **Vue `flush:'sync'` reactivity loops.** InputNumber/InputString/InputTime keep `model Ôćö local Ôćö display` triples synchronized with sync watches (e.g. InputNumber.vue:404+, InputString.vue:36ÔÇô63) and multiSelect registers focus watches with `{flush:'sync'}` (multiSelect.ts:155ÔÇô182). Naive `useState`+`useEffect` will re-order these and cause caret jumps/stale-frame flashes. Port as explicit event-driven updates (call store/`setState` at the event site), not effect chains.
3. **`eval()`-based expression fields** in InputNumber (:349), InputString (:92), InputTime (:220) ÔÇö compiled to `(x, {i})` / `(x, {i, fps})` functions that also drive *other* selected inputs via `multi.update(fn)`. Keep semantics (incl. the `i` index per selected input); mind CSP/lint.
4. **useDrag is load-bearing** for ~10 components: pointer capture on the stable target (child grab-zones unmount mid-drag), pointer-lock with browser-zoom-corrected `movementX/Y`, 3px-mouse/5px-touch thresholds, `shouldDrag` caret-fallthrough. Extract to core first and unit-test; every scrubbing behavior depends on its exact feel.
5. **multiSelect store protocol**: inputs register `{el, focusing, getValue, setValue, confirm, speed}` and receive `{subfocus, index, capture, update, confirm}`; shift-select uses DOM-rect geometry. All value-type inputs (Number/String/Time/Rotary/Switch/ColorPad) participate ÔÇö port the store + one input end-to-end before fanning out.
6. **regl/GLSL** (GlslCanvas + InputColor's pad/slider/wheel `.frag`): one shared offscreen WebGL canvas, PQueue-serialized draws, output blitted via `img.src = canvas.toDataURL()` (slow but simple). Needs `vite-plugin-glsl` kept in the React build. Don't redesign during the port.
7. **Monaco**: theme comes from the theme store (`monacoTheme`) via global `defineTheme`/`setTheme` (MonacoEditor.vue:49); worker config is Vite-implicit. `@monaco-editor/react` loads monaco from CDN by default ÔÇö configure `loader.config({monaco})` to use the bundled `monaco-editor` instead.
8. **Directives & VNode surgery** have no React equivalent: `v-tooltip` (vTooltip.ts, ~10 call sites) Ôćĺ hook/wrapper; InputGroup clones slot children to stamp `inlinePosition/blockPosition` Ôćĺ `Children.map`+`cloneElement` (all inputs must forward those props to their root); Icon/InputDropdown/InputCubicBezier/InputColorPad use `inheritAttrs:false` + `v-bind="$attrs"` Ôćĺ explicit prop spreading.
9. **provide/inject** sites: Tabs (registration keys, symbols.ts), InputColor presets/colorSpace (useInputColor.ts), Timeline scoped slot contract ÔÇö become React contexts / render props.
10. **appConfig `.default` mutability** (extendRef): theme's `setDefault` changes defaults *after* creation and the store reconciles stored-vs-default (appConfig.ts:31ÔÇô41). Replicate in the zustand atom factory or defaults set by `initTweeq` will clobber user-saved values.
11. **Top-layer style islands**: popover content renders outside `.TqViewport`, so PaneModal/MultiSelectPopup re-apply `reset-viewport()` unscoped, and Popover has a `teleport` prop to keep tooltips inside the viewport root. In React, portal into the viewport element (or reproduce the reset) or top-layer content falls back to UA serif.
12. **Actions store lifecycle**: `register()` currently auto-unregisters via `onBeforeUnmount` inside the store call (actions.ts:167) ÔÇö in React it must return a dispose fn used in effect cleanup; bndr keyboard/gamepad emitters must be disposed or hotkeys leak.
13. **Misc**: `-webkit-app-region: drag` swallowing pointer events in TitleBar (noDrag workaround); ParameterGroup's `0frÔćö1fr` grid collapse + transitionend clipping; InputRadio needs `useId` in place of lodash `uniqueId`; Markdown's katex.css is 1137 lines of vendored KaTeX (static asset, don't port); `Icon` supports `char:`/`fill:` prefixes beyond iconify names; InputDrum measures label widths off-screen for cell sizing.
